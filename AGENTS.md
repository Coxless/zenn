# Articles Directory Agent Instructions

このディレクトリはZennに公開する技術記事を管理しています。

## 記事のフォーマット

記事はMarkdown形式で作成し、以下のフロントマターを含める必要があります：

```yaml
---
title: "記事タイトル"
emoji: "絵文字"
type: "tech"  # tech: 技術記事 / idea: アイデア
topics: ["topic1", "topic2"]
published: false  # 下書きはfalse、公開時はtrue
---
```

## 記事作成コマンド

新しい記事を作成する場合：
```bash
pnpm create-article
```

## 画像の参照方法

記事内で画像を使用する場合は、`/images/{記事のslug}/` ディレクトリに配置し、以下の形式で参照：
```markdown
![alt text](/images/{記事のslug}/filename.png)
```

## 記事のスタイルガイド

- 見出しは`##`から開始（`#`は使用しない）
- コードブロックには言語を指定する
- 補足情報は`:::message`ブロックを使用
- 手順は番号付きリストを使用

## プレビュー

記事のプレビューを確認する場合：
```bash
pnpm preview
```

## 文体・トーンガイド

### 基本トーン

- **一人称・カジュアル寄り**の日本語。砕けすぎず、馴れ馴れしくもなく。
- 自分の体験・動機を起点にして書く（「〜しようと思い」「〜に気づきました」等）
- 断言は避け、体験ベースで書く（「〜と思われる」「〜しました」）
- 読者への問いかけで締めることが多い（「コメントください」「フィードバックやPRも歓迎」）

### タイトルパターン

| 記事タイプ | パターン例 |
|-----------|-----------|
| Tips・問題解決 | `[Tips]{問題}ので{解決}した` |
| ツール紹介 | `{ツール名}を作ってみた` |
| 環境構築 | `{環境/ツール}で環境を構築してみた` / `{ツール}が出たので環境を整えてみた` |

### 記事構成テンプレート

#### Tips記事（type: tech）
```
## 概要       ← 何の問題を解決するか1〜2文で
---
## 問題       ← 何が起きたか（環境も記載）
---
## 原因       ← なぜ起きたか（「〜と思われる」でもOK）
---
## 解決方法   ← 具体的な手順・コード
---
## 結果       ← スクショやGIFで確認
---
## 補足       ← 注意点・関連情報
---
## まとめ     ← 1〜2文でまとめ＋他への応用ヒント
```
- セクション間には `---` を挿入する

#### やってみた記事（type: tech）
```
## はじめに   ← 背景・動機
## {ツール名}とは  ← 概要・主な特徴（箇条書き）
## {機能説明} ← 必要に応じてテーブルや図を使う
## 仕組み     ← アーキテクチャ・仕組みの説明（図・コードブロック活用）
## インストール
## 使い方
## 今後の展望
## おわりに   ← 振り返り＋フィードバック歓迎
```

#### 環境構築記事（type: tech）
```
## はじめに   ← 動機・採用した構成の選択肢提示
## 前提環境と構成図  ← 自分の環境スペック
## {ステップ名}  ← 番号付き見出し（### 1. 〇〇）で手順を記述
## まとめと感想  ← メリット箇条書き＋今後の展望
## おわりに   ← 読者への問いかけ
```

### よく使う表現・パターン

```markdown
# GitHubリポジトリはベアURL（リンクテキストなし）
https://github.com/xxx/yyy

# 「〜の様子」をスクショ・GIFで補足
![○○の様子](/images/{slug}/filename.png)

# 手順の前に「↓」で予告する
簡単な例↓

# 状態・コマンド一覧はテーブルで
| 表示 | 状態 | 説明 |
|------|------|------|

# 時点注記・警告は :::message で
:::message
この記事は 2025年XX月XX日時点の内容です。
:::

# まだ未解決・ベータの場合は冒頭 :::message で断り書き
:::message
自分用に作成しており、まだベータ版です。
:::
```

### フロントマターの選び方

- `type: idea` → アイデア系
- `type: tech` → 手順・Tips・環境構築・自作ツール系
- `topics` → 5個以内。ツール名（小文字）＋カテゴリの組み合わせ
- `emoji` → 記事の内容に合ったもの。Rust系は🦀、ツール系は🖥️や🛠️など
